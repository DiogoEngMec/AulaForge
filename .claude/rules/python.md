# Python rules

- Use Python 3.11+.
- Prefer Typer for CLI.
- Use Pydantic models for data contracts.
- Keep modules small and testable.
- Avoid global mutable state.
- Use pathlib instead of string path manipulation.
- Add tests for discovery, ordering, config and checkpoints.
- Use type hints.
- Do not silently swallow exceptions; log them with context.
